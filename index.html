Prework Study Guide

Terminal Commands

    ----- Info -----

  • ls ---> list the directories in the current directory
  • pwd ---> print working directory
  • clear ---> clear the terminal
  • history ---> show the history of commands

    • ----- Navigation -----

  • cd < directory name > ---> change directory
  • cd - ---> change directory to the previous directory
  • cd . ---> change directory to the current directory
  • cd .. ---> change directory to the parent directory
  • cd ../.. ---> change directory to the grandparent directory
  • cd ../../.. ---> change directory to the great grandparent directory
  • cd ~ ---> change directory to home directory
  • cd / ---> change directory to root directory

    • ----- Create & Remove -----

  • mkdir < directory name > ---> make directory
  • rm -rf < directory name > ---> remove a directory
  • touch < file name > ---> create a file
  • rm < file name > ---> remove a file

    • ----- Rename -----

  • mv < file name > < new file name > ---> rename a file
  • mv < directory name > < new directory name > ---> rename a directory

    • ----- Move & Copy -----

  • cp < file name > < new file name > ---> copy a file
  • cp < file name > < directory name > ---> copy a file to a directory
  • cp -r < directory name > < new directory name > ---> copy a directory
  • cp -r < directory name > < directory name > ---> copy a directory to a directory
  • mv < file name > < directory name > ---> move a file to a directory

    • ----- Open -----

  • code . ---> open the current directory in VS Code
  • open < file name > ---> open a file
  • open < file name > -a "application name" ---> open a file in a specific application

HTML

    ----- Page Sections -----

  • < head > ---> contains useful information for browser functions and SEO
  • < body > ---> contains the visual elements on the webpage
  • < header > ---> within the body, appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • < main > ---> within the body, contains the main content of the page
  • < footer > ---> within the body, appears at the bottom of the page and usually contains contact information, site navigation and/or links to social media.
  • < section > ---> defines a section in the document that contains themantic content

    • ----- Elements & Tags -----

  • Header Elements: h1 h2 h3 h4 h5 h6 ---> represent the level of heading a given text block represents.
  • < p > ---> represents a paragraph or block of text
  • < ul > ---> un-ordered list.
  • < ol > ---> ordered list.
  • < li > ---> list item
  • < img > ---> is an image. src attribute contains the image link or path. alt attribute contains alternative text that describes the image. other attributes are available.
  • < br > ---> this creates a line break in between two elements
  • < a > ---> and anchor tag that creates links to other webpages or documents. href attribute declares the link or path to use.

    • ----- Attributes -----

  • id="" ---> gives the element a name
  • class="" ---> assigns the element a css class

    • ----- Tips -----

  • Semantic Element ---> the name of the element describes its content and the role it plays on a webpage.
  • Semantic elements should be prioritised over non-semantic elements such as < div > or < span >.

CSS

    ----- Syntax -----

  • selector {property: value; } ---> this is the structure of CSS
  • CSS selector ---> identifies what to apply stying to.
  • CSS property ---> identifies what styling to apply.
  • CSS value ---> identifies the value of the property.

    • ----- Selectors -----

  • Multiple Selectors ---> apply the same styling to multiple selectors by seperating them by commas: h1, h2, h3 {}
  • * {} ---> This symbol will apply the styling to all the elements on the page.
  • .class-name {} ---> this creates a class. Use class="class-name" inside a HTML tag to apply the class properties.

    • ----- Linking -----

  • < link rel="stylesheet" href="./assets/style.css" > ---> link an external stylesheet to a HTML document to apply the settings. Place in the < head > section.

    • ----- Properties -----

  • display: ; ---> controls how the element is displayed. block elements take up the whole width of the page and inline elements only take up the space they need.
  • width: ; ---> controls the width of the element.
  • height: ; ---> controls the height of the element.
  • margin: ; ---> Pushes the element away from other elements.
  • padding: ; ---> Pushes the border of the element out from the internal element effectivly expanding the size of the whole element.
  • color: ; ---> controls the font colour
  • background-color: ; ---> controls the background colour
  • colour values ---> colour values can be in various forms including names: blue or HEX codes: #ff7c8e or even RGBA: rgba(245, 40, 145, 0.8) where the A is the alpha channel and controls opacity.
  • font-size: ; ---> controls the font size

Git

    Initial Setup & Repo Clone

    ----- Connect Github to Computer Securely -----

  • Follow the instructions to connect your computer to Github via a secure SSH Key
  • git config --global user.name "Your Name" ---> sets your global name in git (should match your username)
  • git config --global user.email "Your Email" ---> sets your global email in git (you can use the hidden email provided)

    • ----- Create a Repository in Github -----

  • In Github create a new repository. Assign yourself as the owner, add any relevant starter files & choose a licence.

    • ----- Clone the Repository to Computer -----

  • In Github copy the SSH clone link.
  • cd ---> Use cd command in terminal to navigate to the folder you want the repository to go.
  • git clone < SSH Link > ---> type this and paste the link to clone the repo to your computer.
  • cd ---> Navigate inside the repo folder if not already.
  • Follow the git flow instructions below to work with the new repo.

    • Git Flow

    ----- Create New Branch -----

  • git checkout main ---> navigate to the main branch
  • git pull origin main ---> updates your main branch with the latest version from github
  • git checkout -b branch/name ---> creates a new branch and navigates to it
  • git status ---> tells you what branch your on and its current state. make sure the branch is clean.

    • ----- Update Current Branch in Git with Our Working Directory -----

  • git add -A ---> adds all the files in our working directory to the staging area of Git
  • git commit -m "short descriptive message" ---> commits all the staged changes to our local repository. The -m flag associates a message with the commit. Commit messages should be short descriptive messages of the changes implemented.

    • ----- Update Github with our Git Branch -----

  • git pull origin main ---> syncs our local branch with the remote branch in GitHub in case a team member has added a new feature to the project. see: "Already up to date"
  • git push origin branch/name ---> updates the Github branch with our local changes

    • ----- Prepare Code in Github for Review -----

  • navigate to the branch and create a pull request ---> This allows others to review your code and comment

    • ----- Merge Complete Code with Main Branch in Github -----

  • merge the pull request ---> when the code is finished then it is time to add it to the main branch by merging

    • ----- Update Your Git Main Branch with the New Changes -----

  • git checkout main ---> navigate to the main branch
  • git pull origin main ---> updates your local main branch with the new merged in feature from github
